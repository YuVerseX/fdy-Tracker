- 状态：`可演示、可本地使用，未生产就绪`

- 当前任务口径：`管理任务已支持 202 异步提交 + phase/progress/heartbeat`
- 当前去重口径：`默认隐藏重复记录，支持历史补齐与管理摘要`
- 当前启动口径：`启动只做 schema/init/seed，不再自动执行历史维护回填`
- 当前健康探针：`/api/health` 已升级为 readiness + scheduler/freshness/task/admin_security 聚合摘要`

## 已知限制

- 还不是多数据源平台
- 结果公示类帖子的岗位口径还在持续收口
- 管理台只有基础 Session 登录，还没有完整用户体系、细粒度权限和监控告警
- 已补单机 VPS Docker 部署基线，但还缺真实 HTTPS 终止、访问控制、监控和 24h 长跑验证
- 历史规则分析 / 规则洞察 / 辅导员口径 / 全量重复治理已改为显式维护动作，不再跟随重启自动执行

## 接下来建议优先做

1. 继续收结果公示类的假岗位与统计口径，降低误提取。
2. 做一次 24h 定时抓取长跑，确认没有 stale heartbeat、freshness 倒退和容器重启循环。
3. 补 HTTPS 与最小访问控制，再评估是否扩展第二数据源。
4. 补监控与告警（任务超时、抓取失败、AI失败率），让长跑更可控。
